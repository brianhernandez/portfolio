About Brian

Hi, my name is Brian. I'm a front end web developer (I make websites) and a Japanese-English translator/interpreter. I graduated from Pace University, New York in 2006 with a B.S. in Computer Science and a B.A. in Modern Languages & Cultures.

After graduating, I lived and worked in Japan for 5 years teaching English to elementary and junior high school students while studying the language and culture of Japan.

Since returning to the U.S. in 2011, I now work as a freelance front end developer working on custom WordPress and Shopify websites. I like to use a bit of my graphic design skills where I can but mainly excel at turning designs into code. I also do a bit of freelance Japanese-English translation work sometimes contributing to large projects involving multinational Japanese investment banks and auto manufacturers.

Some of my hobbies are dancing, martial arts, photography, video editing and laughing. Thanks for stopping by!

Projects

  • Calculator

    Calculator

    The Calculator Project is an attempt to develop a real working calculator in the browser. The project uses an HTML table to layout the various buttons and display field of the calculator. CSS is used for the styling. Vanilla JavaScript is used to listen for click events on the various buttons. Various if and else if statements are used to determine which actions should be taken logically to update the display screen with the entered numeric values and also to appropriately evaluate the mathematically expression. If an expression cannot be evaluated because of improper input, the user is alerted with a pop up dialog box.

  • Forms

    All Form Projects Contain validator.js

    A custom made JavaScript validator file filled with various form input validation functions is used in all of the following form projects. The validator file is named validator.js in the projects’ repositories. This file is required to create the project specific validator file, usually titled [project title]-validate.js, to apply the appropriate validation functions to the corresponding form project.

    Color Builder Form

    The Color Builder Project uses a basic form element to bundle a set of range sliders and a color swatch as input elements. When a click event is detected on any of the range inputs, the background color of the input color swatch is updated with the new value of the corresponding input slider that fired the click event. The rgba() text at the bottom of the color swatch reflects the new RGB and alpha values. The JavaScript script listens for these click events and also attempts to validate the form upon submission, however the range sliders prevent any invalid RGB and alpha inputs.

    Credit Card Form

    The Cred Card Project was an attempt to create a working credit card input form seen on many ecommerce websites. HTML and CSS is used for the structure and styling of the form. Font Awesome icons fonts are pulled in to style the input fields for a better user experience and to signal the type of information required in the form fields. This form contains text and select input elements. The creditcard-validate.js file uses some of the validator functions in validator.js to check form validation upon clicking the submit button. If an invalid input is found, the JavaScript setCustomValidity() function is set with a custom message detailing what type of invalid input was made on the responsible form field.

    Login Form

    The Login Project creates a typical user login UI element. Structured with HTML and CSS, the login form also uses Font Awesome icon fonts for the username and password field icons. A custom styled checkbox input element was also created to simulate a “keep me signed in” toggle. The login-validate.js file uses some functions from the validator.js file to check the input fields for possible invalid entries upon clicking the submit button.

    Questionnaire Form

    The Questionnaire Project creates a simple radio input form to submit one of seven choices with one choice being a custom text input. The HTML form as well as radio and text input elements are used to create the form. CSS is used to customize the look of the radio inputs as well as to add some animations for when the radios are selected/deselected. The questionnaire-validate.js uses some functions from the validator.js file to set the setCustomValidity() function if an invalid input is found upon form submission.

    Scheduling Form

    The Scheduling Project simulates a dentist appointment signup form. HTML is used for the structure of the form, CSS for the styling of the form as well as styling of the form input elements. This form uses a variety of different input elements so the scheduling-validate.js file calls many functions from the validator.js file to check the different types of input formats from the forms fields upon clicking submit. When an invalid input is detected, an appropriate error message is set using the setCustomValidity() function.

    Search Form

    The Search Project simulates a search engine portal. Typical HTML elements are used to create the form. CSS is used extensively to customize the look of various aspects of the two text and select input elements. Functions from validator.js file are used in the search-validate.js to test input validation upon form submission.

    Shipping and Billing Form

    The Shipping and Billing Project creates a simulated shipping and billing form seen on many commerce websites. HTML and CSS create the structure and the styling for the form respectively. The form also uses functions from the validator.js file in the shipping-validate.js to test input validation upon form submission. If an error is detected, the setCustomValidity() function is used to display a custom message describing the type of input error that was found.

    Signup Form

    The Signup Project is a sign up found on most websites. The page is constructed using HTML form and input elements. CSS is used to style the form inputs by making them a little larger than average and also changing the text field background color upon focus. Some functions from the validator.js are used in the signup-validate.js to validate the form field inputs upon submitting the form. If an invalid input is detected, the setCustomValidity() function is set to a message that details the type of input invalidity that was detected.

  • Shopping Cart

    Shopping Cart

    The Shopping Cart Project was my first attempt at making a pure vanilla JavaScript shopping cart complete with functional coupon codes and product quantity update features. Of course, the general structure of the page is made with HTML. Particular attention was made to use semantic HTML5 tags where appropriate. CSS3 is used for the styling and smooth transitions when hovering over different products on the page. Flexbox is also used to make the product-listing grid and the cart spreadsheet area responsive to display appropriately across different screen sizes. JavaScript is used to listen for the different click events on the various buttons across the page. Depending on which button is clicked, the script will take the appropriate action to calculate the totals, change the quantity of products in the cart or apply the appropriate discount amount. Product quantities amounts can also be changed directly by editing the number in the quantity text field and clicking the ‘update’ link below it.

  • Responsive Gallery

    Responsive Gallery

    The Responsive Gallery Project uses a combination of HTML and CSS to create the image gallery affect of one image being selected over another. The images are toggled by unseen radio inputs that are attached to the thumbnail images via label tags. When a an input is checked, a sibling div that provides the space for the larger image to appear in has its background image updated to be the image that gets shown. Flexbox is used to responsively display the thumbnail images along the top of the gallery.

  • Accordion

    Accordion

    The Accordion Project is structured using a ul element. A JavaScript hook is attached within each li element of the unordered list that listens for a click event on it. When a click event is detected, if the target that triggered the event contains the is-active class, the class is removed and its associated is-expanded class attached to its neighboring bellow content div is removed as well. If the is-active class is not attached to the target, attach this class while removing it from all other bellows and also removing the is-expanded class from other bellow content divs. Then, attach this is-expanded class to the sibling bellow content div of the element that triggered the click event. This creates an auto closing and opening of the appropriate bellows of the accordion no matter what state they are found in when clicked.
